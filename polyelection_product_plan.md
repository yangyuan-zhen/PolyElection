# 🗳️ PolyElection — 预测市场选举量化分析平台

> **产品定位**：为 Polymarket / Kalshi 等预测市场的选举合约交易者，提供数据驱动的决策工具。
>
> **一句话卖点**：当民调说 A 领先 5%，但赔率只给 52%——谁对了？我们帮你算。

### 2.2 🧬 PEB 算法（Poll Ensemble Blending）

**PEB 算法逻辑**：

- **多源输入**: 整合多个民调机构的原始数据 `{YouGov: 48%, Ipsos: 51%, ...}`。
- **权重修正**: 根据每个机构的历史准确度（MAE）和政治倾向（Biased Index）动态调整权重。
- **融合输出**: 算出最接近真实的融合支持率。

**权重计算逻辑**：

1. 收集民调机构在同类历史选举中的偏差
2. 计算每个机构的 MAE (Mean Absolute Error)
3. 用 MAE 倒数作为权重（误差越小 → 权重越大）
4. 冷启动期（无历史的新机构）使用等权平均

### 2.5 🤖 AI 选情洞察 (基于 Groq/LLaMA)

定制选举版 P0→P4 分析链：

```
P0: 事件冲击检测 (候选人退选/丑闻/突发政策)
    → "重大事件" 自动触发分析刷新
P1: 民调动向节奏
    → 连续3个民调同向 → 趋势确认
    → 民调方向反转 → 发出信号
P2: 市场与民调偏离分析
    → 偏离 > 5% → 可能套利机会
    → 偏离收敛中 → 市场正在修正
P3: PEB 融合概率 vs 市场定价
    → 概率差 → 交易建议 (做多/做空/观望)
P4: 历史基准
    → 同类选举的历史赔率走势参考
```

### 2.6 🔔 冲击评分 (Shock Score)

```python
# 检测赔率异常波动
shock_score = 0.0

# 1. 赔率突变 (权重 0.4)
if abs(odds_change_1h) > 3%:
    shock_score += min(abs(odds_change_1h) / 10, 1.0) * 0.4

# 2. 交易量激增 (权重 0.35)
if volume_1h > avg_volume * 3:
    shock_score += min(volume_1h / (avg_volume * 10), 1.0) * 0.35

# 3. 多市场赔率同步变动 (权重 0.25)
if polymarket_move * kalshi_move > 0:  # 同向
    shock_score += 0.25

# shock_score > 0.5 → 触发 AI 事件分析
```

## 4. 技术架构

### 4.2 核心技术架构 (前后端分离)

- **Backend**: Python / FastAPI (数据中心)
- **Frontend**: React / Vite / Tailwind CSS (高性能渲染)
- **Communication**: REST API + WebSocket (实时行情)
- **Task Runner**: APScheduler / Redis (后台采集任务)
